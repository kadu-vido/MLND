Questão 1 - Observação de Atributos¶

Para lembrar, estamos utilizando três atributos do conjunto de dados dos imóveis de Boston: 'RM','LSTAT'e'PTRATIO'`. Para cada observação de dados (vizinhança):

  • 'RM' é o número médio de quartos entre os imóveis na vizinhança.
  • 'LSTAT' é a porcentagem de proprietários na vizinhança considerados de "classe baixa" (proletariado).
  • 'PTRATIO' é a razão de estudantes para professores nas escolas de ensino fundamental e médio na vizinhança.

Utilizando sua intuição, para cada um dos atributos acima, você acha que um aumento no seu valor poderia levar a um aumento no valor do 'MEDV' ou uma diminuição do valor do 'MEDV'? Justifique sua opinião para cada uma das opções.
Dica: Você espera que um imóvel que tem um valor 'RM' de 6 custe mais ou menos que um imóvel com valor 'RM' de 7?

Resposta:

  • O número médio de cômodos, RM, é um indicador do tamanho médio dos imóveis na vizinhança. Normalmente, o tamanho das casas dentro de uma região é aproximadamente uniforme (não é habitual encontrar bairros com mansões ao lado de chalés). O preço de um imóvel cresce com seu tamanho, então RM e MEDV são diretamente proporcionais.
  • LSTAT está inversamente relacionado ao poder aquisitivo dos habitantes de uma região: se o valor de LSTAT é alto, predominam pessoas de baixo poder aquisitivo. É razoável supor que o valor dos imóveis cresce com o poder aquisitivo dos habitantes. Assim sendo, LSTAT e MEDV são inversamente proporcionais.
  • Valores altos de PTRATIO indicam salas de aula lotada. Isso normalmente é prejudicial para os estudantes, então é de se supor que se uma escola tivesse mais dinheiro, ela contrataria mais professores para evitar essa situação. Além disso, parece mais provável que escolas com mais dinheiro fiquem em regiões mais ricas. Dessa maneira, PTRATIO provavelmente se correlaciona positivamente a MDEV.

Implementação: Definir uma Métrica de Desempenho¶

É difícil medir a qualidade de um modelo dado sem quantificar seu desempenho durante o treinamento e teste. Isso é geralmente feito utilizando algum tipo de métrica de desempenho, através do cálculo de algum tipo de erro, qualidade de ajuste, ou qualquer outra medida útil. Para este projeto, você irá calcular o coeficiente de determinação, R2, para quantificar o desempenho do seu modelo. O coeficiente da determinação para um modelo é uma estatística útil em análise regressa, como se ele frequentemente descrevesse como "good" a capacidade do modelo de fazer estimativas.

Os valores para R2 têm um alcance de 0 a 1, que captura a porcentagem da correlação ao quadrado entre a estimativa e o valor atual da variável alvo. Um modelo R2 de valor 0 sempre falha ao estimar a variável alvo, enquanto que um modelo R2 de valor 1, estima perfeitamente a variável alvo. Qualquer valor entre 0 e 1 indica qual a porcentagem da variável alvo, ao utilizar esse modelo, ele pode ser explicado pelos atributos. Um modelo pode dar também um R2 negativo, que indica que o modelo não é melhor do que aquele que estima ingenuamente a média da variável alvo.

Para a função ‘performance_metric’ na célula de código abaixo, você irá precisar implementar o seguinte:

  • Utilizar o r2_score do sklearn.metrics para executar um cálculo de desempenho entre y_true e y_predict.
  • Atribuir a pontuação do desempenho para a variável score.

from sklearn.metrics import r2_score

def performance_metric(y_true, y_predict):
    """ Calcular e retornar a pontuação de desempenho entre 
        valores reais e estimados baseado na métrica escolhida. """
    
    if len(y_true) == len(y_predict):
        return r2_score(y_true, y_predict)
    print "Os tamanhos dos vetores de entrada não são iguais"
    print "y_true tem tamanho %d" % (len(y_true))
    print "y_predict tem tamanho %d" % (len(y_predict))
    return 0

Resposta: A busca em matriz consiste em criar uma matriz com todas as combinações possíveis de parâmetros, criar uma instância do modelo com cada uma delas e fazer a avaliação para saber qual combinação leva às melhores previsões.

Questão 8 - Validação Cruzada¶

O que é a técnica de treinamento de validação-cruzada k-fold? Quais benefícios essa técnica proporciona para busca em matriz ao otimizar um modelo?
Dica: Assim como há um raciocínio por trás de utilizar um conjunto de teste, o que poderia dar errado ao utilizar busca em matriz sem um conjunto de validação cruzada?

Resposta: K-fold é uma técnica que particiona o conjunto de dados disponível em K subconjuntos de mesmo tamanho, utilizando K-1 para treinamento e o restante para teste. O processo é repetido K vezes, de maneira que cada subconjunto é utilizado uma vez para teste. Podemos utilizar k-fold para criar conjuntos de treinamento e teste para cada instância do modelo propostos pela busca em matriz. Isso permite que evitemos instâncias que sofram de overfitting.

Implementação: Ajustar um Modelo¶

Na sua última implementação, você vai precisar unir tudo o que foi aprendido e treinar um modelo utilizando o algoritmo de árvore de decisão. Para garantir que você está produzindo um modelo otimizado, você treinará o modelo utilizando busca em matriz para otimizar o parâmetro de profundidade máxima ('max_depth') para uma árvore de decisão. Esse parâmetro pode ser entendido como o número de perguntas que o algoritmo de árvore de decisão pode fazer sobre os dados antes de fazer uma estimativa. Árvores de decisão são parte de uma classe de algoritmos chamados algoritmos de aprendizagem supervisionada.

Para a função fit_model na célula de código abaixo, você vai precisar implementar o seguinte:

  • Utilize o DecisionTreeRegressor do sklearn.tree para gerar um objeto regressor de árvore de decisão.
    • Atribua esse objeto à variável 'regressor'.
  • Gere um dicionário para 'max_depth' com os valores de 1 a 10 e atribua isso para a variável 'params'.
  • Utilize o make_scorer do sklearn.metrics para gerar um objeto de função de pontuação.
    • Passe a função performance_metric como um parâmetro para esse objeto.
    • Atribua a função de pontuação à variável 'scoring_fnc'.
  • Utilize o GridSearchCV do sklearn.grid_search para gerar um objeto de busca por matriz.

    • Atribua o objeto GridSearchCV para a variável 'grid'.

from sklearn.tree import DecisionTreeRegressor as dtr
from sklearn.metrics import make_scorer
from sklearn.grid_search import GridSearchCV as gs

def fit_model(X, y):
    """ Desempenhar busca em matriz sobre o parâmetro the 'max_depth' para uma 
        árvore de decisão de regressão treinada nos dados de entrada [X, y]. """
    
    # Gerar conjuntos de validação-cruzada para o treinamento de dados
    cv_sets = ShuffleSplit(X.shape[0], n_iter = 10, test_size = 0.20, random_state = 12345)

    regressor = dtr(random_state = 12345)
    params = {'max_depth': range(1,11)}
    scoring_fnc = make_scorer(performance_metric)
    grid = gs(regressor, params, scoring_fnc, cv=cv_sets)

    # Ajustar o objeto de busca em matriz com os dados para calcular o modelo ótimo
    grid = grid.fit(X, y)

    # Devolver o modelo ótimo depois de realizar o ajuste dos dados
    return grid.best_estimator_

Sensibilidade¶

Um modelo ótimo não é necessariamente um modelo robusto. Às vezes, um modelo é muito complexo ou muito simples para generalizar os novos dados. Às vezes, o modelo pode utilizar um algoritmo de aprendizagem que não é apropriado para a estrutura de dados especificado. Outras vezes, os próprios dados podem ter informação excessiva ou exemplos insuficientes para permitir que o modelo apreenda a variável alvo – ou seja, o modelo não pode ser ajustado. Questão 11 - Aplicabilidade¶

Em poucas linhas, argumente se o modelo construído deve ou não ser utilizado de acordo com as configurações do mundo real.
Dica: Algumas questões para responder:

  • Quão relevante dados coletados em 1978 podem ser nos dias de hoje?
  • Os atributos presentes são suficientes para descrever um imóvel?
  • Esse modelo é robusto o suficiente para fazer estimativas consistentes?
  • Dados coletados em uma cidade urbana como Boston podem ser aplicados para uma cidade rural?

Resposta:

  • É difícil saber se os dados de 1978 continuam relevantes: por um lado, eles foram escalonados para compensar a inflação, mas por outro lado eu não sei se a cidade não passou por grandes renovações ou mudanças de perfil de imóvel. Uma região que tenha passado por gentrificação ou a popularização recente de studios (apartamentos com número reduzido de cômodos), por exemplo, podem mudar bastante o perfil dos dados.
  • Os atributos presentes não parecem ser suficientes para descrever um imóvel. Eles ignoram diversos fatores que acredito serem relevantes, como taxas de criminalidade, acesso à rede de transporte público, proximidade e qualidade de hospitais etc.
  • O modelo parece razoavelmente robusto, considerando os atributos: na seção acima a variação de preço de um imóvel estimado em cerca de 411 mil é de cerca de 30 mil em 10 tentativas. Isso é uma variação de aproximadamente 7%.*
  • Aplicar dados de uma cidade em outra parece contra-producente, já que o mercado de imóveis varia enormemente entre cidades. Isso parece especialmente ruim se as cidades têm perfis diferentes (urbana vs rural, por exemplo), já que os atributos relevantes podem variar. Mesmo o conjunto relativamente pobre de atributos usado talvez tenha valores diferentes: é possível que em uma cidade rural, o número médio de quartos seja mais alto, se predominarem estruturas familiares diferentes (com três gerações vivendo na mesma casa ou maior número de filhos por casal, por exemplo).

Dados os fatores acima, não me parece razoável utilizar o modelo construído neste exercício para estimativas de valor de imóveis em uma cidade real nos dias de hoje. Provavelmente, também seria possível construir um modelo melhor para os valores da casa em 1978 se forem incluídos mais atributos—e a relevância dos mesmos pode ser medida de acordo com seu poder preditivo.
